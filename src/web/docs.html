Clai Docs: Cloud + ClaiStore + Ops

This docs portal is designed for production operators. It focuses on server-first installs, cloud deployment, and release-safe operations.

server-first cloud-ready never install on viewer machine

ClaiStore Server Install Flow

Step 1

User clicks Install on Server

The dashboard triggers a POST request to /api/community/install/{item_id} with CSRF protection.

Step 2

Server stores package under runtime data

Package bytes are saved in community_hub/installed inside the TALOS data directory on the host.

Step 3

OpenClaw manifests are imported automatically

If the package looks like an OpenClaw manifest, tools and skill docs are imported into the runtime registry/docs.

Security default: install only stores/imports artifacts. It does not auto-execute arbitrary code from packages.

Runtime Paths

Community Packages
community_hub/packages
Installed Artifacts
community_hub/installed
Dynamic Tools Registry
projects/dynamic_tools.json
Skill Docs Directory
tools/

Cloud Deployment Baseline (Ubuntu VM)

Install from release package

sudo apt update
sudo apt install -y curl ca-certificates
curl -L -o talos_1.0.0_amd64.deb https://github.com/<org>/<repo>/releases/download/v1.0.0/talos_1.0.0_amd64.deb
sudo apt install -y ./talos_1.0.0_amd64.deb
sudo systemctl enable clai-talos
sudo systemctl restart clai-talos
sudo systemctl status clai-talos --no-pager

Set production environment keys

sudo tee -a /var/lib/clai-talos/.env >/dev/null <<'EOF'
WEB_PORT=8080
BOT_NAME=Clai-TALOS
TELEGRAM_BOT_TOKEN=REPLACE_ME
OPENAI_API_KEY=REPLACE_ME
DASHBOARD_REMOTE_URL=https://your-user.github.io/your-repo/dashboard-fragment.html
EOF
sudo chown clai-talos:clai-talos /var/lib/clai-talos/.env
sudo systemctl restart clai-talos

Reverse proxy + TLS (Nginx sketch)

sudo apt install -y nginx certbot python3-certbot-nginx
sudo certbot --nginx -d your.domain.example
# Proxy pass to http://127.0.0.1:8080 in nginx site config
sudo systemctl reload nginx

GitHub Docs Publishing

Publish this docs style on GitHub Pages for external read-only docs. Keep in-app docs at /docs for authenticated operators. 

# from repo root
mkdir -p docs-site
# put your static docs site in docs-site/
git add docs-site
git commit -m "Add docs website"
git push
# Enable GitHub Pages from docs-site branch/folder in repo settings

Tip: keep operational secrets out of public docs pages. Public docs should contain placeholders only.
